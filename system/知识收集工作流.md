# 知识收集工作流

> 外部信息收集 → Inbox待处理 → 人工采纳 → 分类归档 → 去重整理 → 沉淀入库

## 流程概览

```
外部扫描 → Inbox/待处理 → 人工审阅 → 分类归档 → 去重检查 → 写入知识库
    ↑                                                        ↓
    └──────────── 每周自动整理 ←──────────────────────────────┘
```

## 1. Inbox 待处理区

路径：知识库/Inbox/

所有外部扫描到的信息先放这里。每个条目不超过5句话，必须有关联标签。

## 2. 人工审阅

- 每天查看Inbox，标记采纳/忽略
- 采纳 → 移动到对应分类目录
- 忽略 → 标记 dismissed

## 3. 分类归档

| 类别 | 目标目录 |
|------|---------|
| 开发（新工具/框架） | 知识库/实体/ |
| 开发（方法论/架构） | 知识库/概念/ |
| 设计（UI/UX） | 知识库/概念/ |
| 企业AI（行业案例） | 知识库/概念/ |
| 对比分析 | 知识库/对比分析/ |
| 主题综述 | 知识库/主题综述/ |

## 4. 去重检查

- 标题相似度 > 80% → 合并或标记重复
- 已有更好描述 → 补充链接到旧笔记
- 已有更差 → 替换旧笔记

## 5. 写入知识库

- 更新 index.md 注册新页面
- 建立双向链接
- 写入知识库日志

## 6. 每周自动整理

每周一自动执行：
- 扫描Inbox中所有 pending 条目
- 搜索现有知识库是否有重复
- 有重复 → 合并或链接
- 无重复 → 创建新笔记
- 生成周度知识报告
