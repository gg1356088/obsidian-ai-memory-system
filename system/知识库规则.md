# 知识库规则

本文件定义知识库的维护方式。

## 一、整体原则

- 原始资料放在知识库/原始资料，默认只读
- 正式知识放在知识库/和日志/中
- 优先更新已有页面，而非盲目新建
- 重要分析写回知识库，不只留聊天里
- 不同 AI agent 通过读取独立 .md 笔记进行参考性学习，不互相覆盖

## 二、知识库八大分类

| 分类 | 路径 | 用途 |
|------|------|------|
| 索引 | 索引/ | 跨分类串联枢纽 |
| 人物 | 人物/ | 联系人、团队成员 |
| 项目 | 项目/ | 进行中的项目与工作流 |
| 决策 | 决策/ | 重要决策记录 |
| 公司 | 公司/ | 客户公司、合作方 |
| 会议 | 会议/ | 会议记录与纪要 |
| 日志 | 日志/ | 时间日志与会话沉淀 |
| 知识库 | 知识库/ | 实体、概念、综述、对比、问题 |

另有系统/存放规则、规范、模板等元信息。

## 三、知识库子类型

| 类型 | 路径 | 适合 |
|------|------|------|
| 来源摘要 | 知识库/来源摘要 | 文章、PDF、播客 |
| 实体页 | 知识库/实体 | 工具、产品、项目 |
| 概念页 | 知识库/概念 | 理论、方法、术语 |
| 主题综述 | 知识库/主题综述 | 长期研究主题 |
| 对比分析 | 知识库/对比分析 | 方法对比、工具对比 |
| 问题与待研究 | 知识库/问题与待研究 | 未解决问题 |
| 会话沉淀 | 日志/会话沉淀 | 高价值结论 |

## 四、文件命名规范

推荐格式：YYYY-MM-DD 项目或主题 类型.md

- 时间优先，便于按阶段追溯
- 包含主题、实体或核心对象
- 包含页面类型（摘要、沉淀、规则、综述）
- 避免只用泛名（总结.md、记录.md）

## 五、写作要求

- 一条页面只讲一个清晰对象
- 用双链把相关页面串起来
- 不确定内容标记待验证
- 补齐 frontmatter 字段，见 [[系统/字段规范]]

## 六、主动沉淀规则

处理方式（按八大分类）：

1. 联系人 → 人物/
2. 项目 → 项目/
3. 决策 → 决策/
4. 公司 → 公司/
5. 会议 → 会议/
6. 日志 → 日志/
7. 跨分类 → 索引/
8. 原始资料 → 知识库/原始资料
9. 资料摘要 → 知识库/来源摘要
10. 工具产品 → 知识库/实体
11. 方法概念 → 知识库/概念
12. 研究方向 → 知识库/主题综述
13. 对比判断 → 知识库/对比分析
14. 不确定 → 知识库/问题与待研究

## 七、AI 优先设计原则

默认假设读者是另一个 AI agent：

1. 指令层与人读层分离
2. 纯 Markdown，跨 AI 工具兼容
3. 触发条件可机器判断
4. 动作有验证方式
5. 部署指令内置
