# 图谱化读取说明

这个知识库虽然不是 CodeGraph 那种代码索引系统，但后续会按"知识图谱"的思路来读取。

## 我会怎么读

读取时优先看这几层：

1. 目录层
   - 页面属于哪个类型
   - 在知识库里处于哪一层

2. 元数据层
   - `type`
   - `status`
   - `source_notes`
   - `related_*`

3. 双链层
   - `[[知识库/概念/双链]]`
   - 页面之间的显式连接

4. 正文层
   - 核心定义
   - 阶段性判断
   - 争议点
   - 待验证内容

## 理想读取链路

最常见的追踪方式是：

- 来源摘要 -> 概念页
- 来源摘要 -> 实体页
- 概念页 -> 主题综述
- 实体页 -> 主题综述
- 主题综述 -> 开放问题

## 为什么要这样设计

这样做的目标是避免：

- 笔记只是一堆碎片
- 有内容但找不到关系
- 有结论但忘了来自哪里
- 有问题但没有挂到主题上

## 你后续能期待什么

当这套关系逐渐补起来后，我以后更容易回答这类问题：

- 某个概念最早是从哪些来源进入知识库的
- 某个主题目前依赖哪些来源和概念
- 哪些实体是一个主题里的中心节点
- 目前哪些问题还没有闭环
