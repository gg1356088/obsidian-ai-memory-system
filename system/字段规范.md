# 字段规范

这个文件定义各类页面建议使用的 frontmatter 字段。目标不是机械填表，而是让后续读取关系更稳定。

## 通用字段

所有正式页面尽量包含：

```yaml
---
title: 页面标题
type: source-summary | entity | concept | theme | comparison | question | conversation
status: seed | active | stable | archived
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - knowledge-base
场景关键词: [frontmatter 字段 元信息 双链 关系字段 title type status tags]
aliases: []
source_notes: []
related_entities: []
related_concepts: []
related_themes: []
related_questions: []
---
```

## 来源摘要页

推荐额外字段：

```yaml
source_type: article | pdf | video | podcast | meeting | note
source_author: ""
source_url: ""
source_date: YYYY-MM-DD
confidence: high | medium | low
```

## 实体页

推荐额外字段：

```yaml
entity_type: person | company | product | tool | project
organizations: []
people: []
timeline_anchor: ""
```

## 概念页

推荐额外字段：

```yaml
concept_type: method | term | framework | model | workflow
opposing_views: []
common_misunderstandings: []
```

## 主题综述页

推荐额外字段：

```yaml
scope: ""
key_claims: []
open_loops: []
```

## 对比分析页

推荐额外字段：

```yaml
comparison_targets: []
decision_context: ""
recommended_choice: ""
```

## 问题页

推荐额外字段：

```yaml
question_state: open | researching | answered | parked
next_action: ""
```

## 会话沉淀页

推荐额外字段：

```yaml
conversation_origin: ""
decision_points: []
```

## 关系字段怎么理解

- `source_notes`：这页主要来自哪些来源摘要
- `related_entities`：相关的人、公司、工具、项目
- `related_concepts`：相关方法、术语、框架
- `related_themes`：属于哪个大主题
- `related_questions`：这页还牵连哪些待解决问题

> [!note]
> 后续我在读取你的知识库时，会优先把这些字段当作"显式关系"，再结合正文里的 `[[知识库/概念/双链]]` 一起理解。
