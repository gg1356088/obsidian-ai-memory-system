# 知识库自进化机制 v2.1

知识库不是静态 wiki——它应该在 AI 会话中自动进化，同时通过定时任务自行运转，**不依赖人类主动打开**。

## 一、核心原则

1. **双引擎驱动** — AI 会话 + 定时任务并行
2. **自动触发，不等人喊** — 会话开始自动检查，定时到点自动执行
3. **渐进式，不追求完美** — 每次进化一点
4. **可追溯** — 每次进化都记录在 `日志/每日进化/` 中

## 二、触发时机

### 引擎一：AI 会话驱动

| 时机 | 做什么 | 深度 |
|------|--------|------|
| 每次会话开始 | 快速健康扫描（断链、孤儿、空分类） | 1-2 分钟 |
| 发现新信息时 | 自动分类写入对应目录 | 实时 |
| 会话结束时 | 沉淀当次会话价值内容 | 3-5 分钟 |

### 引擎二：定时任务驱动

| 频率 | 做什么 | 深度 |
|------|--------|------|
| 每天 | 健康扫描 + gstack learnings 同步 | 轻量 |
| 每周 | 深度整理（链接密度、重复内容、趋势分析） | 中量 |
| 每月 | 结构审查（分类合理性、过时清理） | 重量 |

## 三、每日进化检查清单

### 3.1 健康扫描
- [ ] 断链检查：wikilink 是否指向不存在的文件？
- [ ] 孤儿检查：是否有页面没有任何 inbound link？
- [ ] Frontmatter 检查：新页面是否缺少 title/type/status/tags？
- [ ] 空分类检查：是否有分类长期无内容？
- [ ] 错误模式匹配：读取 `系统/AI错误模式库.md` 场景关键词，匹配当前任务
- [ ] gstack learnings 同步：读取 learnings.jsonl，比对错误模式库

### 3.2 内容增长
- [ ] 新工具、产品、人物 → 创建实体页
- [ ] 新方法、概念、术语 → 创建概念页
- [ ] 重要决策 → 写入 `决策/`
- [ ] 外部资料 → 写入 `知识库/Inbox/`

### 3.3 关系织网（每周）
- [ ] 扫描内容相似但未链接的页面 → 添加双向链接
- [ ] 更新实体地图、概念地图

### 3.4 结构优化（每月）
- [ ] 页面该合并/拆分？
- [ ] 分类体系是否合理？
- [ ] 知识缺口盘点

## 四、进化维度

| 维度 | 目标 |
|------|------|
| 健康维护 | 链接不坏、结构清晰、元信息完整 |
| 内容增长 | 从会话和外部资料中持续提取新知 |
| 关系织网 | 新页面至少链接 2 个已有页面 |
| 结构优化 | 调整分类、合并重复 |
| 时效管理 | 标记过时内容 |
| 跨系统同步 | gstack learnings ↔ Obsidian 错误模式库 |

## 五、自动写入规则

| 遇到什么 | 写入哪里 | 条件 |
|----------|----------|------|
| 新的工具/产品/服务 | `知识库/实体/` | 用户提到并使用 |
| 新的方法/流程/概念 | `知识库/概念/` | 可复用 |
| 重要决策 | `决策/` | 影响后续工作 |
| 联系人信息 | `人物/` | 新的合作伙伴 |
| 外部文章/资源 | `知识库/Inbox/` | 高价值 |
| 会话中的关键结论 | `日志/会话沉淀/` | 可复用 |
| gstack learning 新条目 | `系统/AI错误模式库.md` | Obsidian 中无对应条目 |

## 六、内容过滤规则

知识库不是聊天记录。写入前必须过滤，**只保留三类内容**：

| 保留 | 丢弃 |
|------|------|
| **决策** — 影响后续工作方向的选择 | 口水词、寒暄 |
| **承诺** — 对用户/项目的明确承诺 | 过程性讨论 |
| **核心洞察** — 可复用的发现、规律 | 一次性操作细节 |

**判断标准：三个月后回看，这条信息还有价值吗？**

## 七、每日进化日志

写入 `日志/每日进化/YYYY-MM-DD.md`：

```markdown
---
date: YYYY-MM-DD
type: daily-evolution
---

# 每日进化 YYYY-MM-DD

## 健康扫描
- 断链：0 / 孤儿：0
- gstack 同步：0/0 已覆盖
- 问题：无

## 新增内容
- ...

## 待处理
- ...
```
