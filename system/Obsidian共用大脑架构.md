# Obsidian 共用大脑架构

Obsidian 是所有 AI 工具、所有项目的**长期知识中枢**。各项目 AGENTS.md 只是轻量入口。

## 一、核心设计：一个大脑，多个入口

```
                    ┌─────────────────────────┐
                    │   Obsidian Vault         │
                    │   （长期知识中枢）          │
                    │                          │
                    │   系统/  人物/  决策/       │
                    │   项目/  日志/  知识库/     │
                    │   公司/  会议/  索引/       │
                    │                          │
                    │   所有 AI 工具共读共写       │
                    └──────┬──────────┬────────┘
                           │          │
              ┌────────────┘          └────────────┐
              ▼                                    ▼
    ┌──────────────────┐              ┌──────────────────┐
    │ 项目 A            │              │ 项目 B            │
    │ AGENTS.md        │              │ AGENTS.md        │
    │ （轻量入口）      │              │ （轻量入口）      │
    │ → 引用 Obsidian  │              │ → 引用 Obsidian  │
    └──────────────────┘              └──────────────────┘
```

**一句话：Obsidian 是大脑，各项目的 AGENTS.md 只是手指。** 手指不同，但指的都是同一个大脑。

## 二、三条硬规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | **所有长期知识写入 Obsidian** | 不是聊天记录、不是项目 README、不是临时的 AGENTS.md。Obsidian 是唯一的长期存储。 |
| 2 | **不同 AI 工具、不同会话，读同一份 Obsidian** | Claude Code 写入的人物页，Codex 下次会话能读到。Codex 写入的决策页，Cursor 下次会话能读到。不互相覆盖，各自追加。 |
| 3 | **各项目 AGENTS.md 只放入口和引用，不放知识本体** | AGENTS.md 说"去 Obsidian 读 X 规则"，而不是把规则复制一份到 AGENTS.md 里。入口文件是手指，Obsidian 是大脑。 |

## 三、为什么必须这样设计

| 如果不用共用大脑 | 用共用大脑 |
|----------------|----------|
| 每个项目各自存一份知识 → 重复、矛盾、过时 | 所有项目读同一份知识 → 一处更新，处处生效 |
| Claude Code 学到的东西，Codex 不知道 | 不同 AI 工具通过同一个 Obsidian Vault 共享知识 |
| 换个 AI 工具，经验全部丢失 | 不管用什么工具，都从 Obsidian 读取和写入 |
| 知识跟着项目走，项目完了知识没了 | 知识跟着 Obsidian 走，项目是临时的，知识是永久的 |

## 四、实际效果

```
场景：你在 Claude Code 里发现了一个新工具，AI 自动登记到 Obsidian 知识库/实体/

第二天，你换 Codex 做另一个项目
→ Codex 读到 Obsidian 里的实体目录，知道有什么工具可以用
→ 不需要你重新解释

一周后，你在 Cursor 里写网站代码
→ Cursor 读到 Obsidian 里的人物/团队结构，知道给谁看
→ 读到决策/运营规则，知道什么内容不能放
→ 不需要你重新交代背景
```

## 五、不同会话、不同 AI Agent 如何协作

- 后续有价值的更新**默认写入 Obsidian**
- 不同会话、不同 AI agent 通过读取相关独立 `.md` 笔记进行**参考性学习**
- **不互相覆盖**，不合并成单一总文件
- 各工作区的 `.md` 文件保持独立维护、按职责优化，但可以**互相参考**
- 共用原则、重要经验、可复用方法统一沉淀到 Obsidian
- 如果某条规则适合多个工作区，**先写进 Obsidian**，再由各工作区按需引用

## 六、部署方式

**Obsidian Vault 位置：** 独立目录，不在任何项目下面。例如：

```
~/obsidian/                        ← 共用大脑
~/project-a/                       ← 项目 A（有自己的 AGENTS.md）
~/project-b/                       ← 项目 B（有自己的 AGENTS.md）
```

**各项目的 AGENTS.md 引用 Obsidian：**

```markdown
## 知识库规则

读取并执行 `~/obsidian/系统/AI入口.md`。
读取并执行 `~/obsidian/系统/AI协作准则.md`（含反问机制）。
读取并执行 `~/obsidian/系统/自进化机制.md`。

写入规则：所有长期知识写入 Obsidian（`~/obsidian/`），不写入本项目目录。
```

> ⚠️ 各项目 AGENTS.md 引用 Obsidian 时用**绝对路径**。相对路径在不同项目下会指到不同位置。
